# MissNet UI/UX 重构蓝图 v1

> 目标对标：Google Drive App（信息架构、列表效率、状态反馈、成熟产品感）

## 1. 项目目标

本次重构聚焦四条主线：

1. UI 视觉系统统一
2. UX 信息架构重组
3. 核心页面重构（Home / Library-Downloads / Player / Search / Settings）
4. bug 修复与体验打磨同步推进

### 暂不优先

- 真正离线 HLS 导出完整版
- 大规模新功能扩张
- 纯装饰性视觉实验


## 1.1 首轮修正（Gemini 审阅后）

基于外部审阅反馈，重构蓝图补充以下修正原则：

1. **Home 首轮定位改为“任务恢复入口”**，而不是泛化的展示型首页
   - 首屏优先：继续播放、最近访问、进入资产中心
   - 推荐/发现内容后置
2. **Library / Downloads 首轮定位改为“任务与资产状态中心”**
   - 优先处理：进行中任务、失败恢复、已完成快速访问
   - 排序/筛选不是首轮核心目标
3. **Player 首轮仅做轻量收口**
   - 优先信息层级与状态可见化
   - 暂不做大规模交互创新
4. **状态系统首轮实现先收敛**
   - 先覆盖关键路径，避免 UI 状态组合爆炸
5. **动画策略首轮以克制为主**
   - 优先采用 Material 默认 motion
   - 不做重表达型共享动画和过度交互动效
6. **错误恢复优先做“可理解 + 可恢复”**
   - 更好的失败文案
   - 一键重试 / 恢复
   - 自动化重试放到后续阶段

## 2. 北极星目标

把 MissNet 从“功能逐步堆起来的 App”升级为：

- 结构清晰
- 状态稳定
- 长期使用不累
- 接近成熟一线产品体验

## 3. 对标 GDrive 的关键原则

### 3.1 内容优先
- 内容 > 装饰
- 层级 > 炫技
- 结构 > 花活

### 3.2 克制且系统化
- 少颜色噪音
- 少大面积装饰
- 多用统一组件、统一间距、统一操作逻辑

### 3.3 状态明确
必须统一表达：
- 下载中
- 等待导出
- 导出中
- 已导出
- 导出失败
- 不支持
- 加载中 / 空状态 / 错误态

### 3.4 可预测
用户要能快速理解：
- 现在在哪
- 当前状态是什么
- 下一步能做什么

## 4. 信息架构蓝图

### 一级结构建议
1. Home
2. Library / Downloads
3. Search
4. Settings / Profile

### 页面角色

#### Home
- 内容入口页
- 继续浏览 / 最近使用 / 推荐内容
- 不是纯展示海报页

#### Library / Downloads
- 用户资产中心
- 下载、保存、导出内容统一管理
- 不是单纯下载记录页

#### Player / Detail
- 核心内容体验页
- 播放、详情、下载/导出/保存闭环

#### Search
- 一级能力
- 支持历史、建议、结果过滤

#### Settings
- 播放、缓存、导出、偏好与关于

## 6. 核心页面蓝图

### 6.1 Home
目标：作为“任务恢复入口页”，优先服务继续播放、最近访问和快速回到资产中心。

建议结构：
1. Header（标题 / 搜索 / 菜单）
2. 最近 / 继续区域（继续播放 / 最近访问 / 最近完成任务）
3. 快速入口（进入资源库 / 下载管理 / 最近导出）
4. 次级内容区（推荐 / 最近更新 / 常看，后置）

关键要求：
- 首屏快速理解
- 减少无效装饰
- 强化“继续”和“最近”而非纯轮播

### 6.2 Library / Downloads
目标：从功能页升级为“任务与资产状态中心”。

建议结构：
1. Header（标题 / 搜索）
2. 当前任务区（下载中 / 导出中 / 失败待恢复）
3. 已完成区（最近完成 / 已导出 / 可快速打开）
4. 增强层（排序 / 筛选 / overflow / 批量操作，后续）

关键要求：
- 列表扫描效率高
- 状态表达统一
- 失败可恢复
- 动作层级清晰

### 6.3 Player / Detail
目标：首轮先完成轻量收口，优先解决信息层级与状态可见化。

建议结构：
1. 顶栏（返回 / 标题 / 更多）
2. 主内容区（视频/封面 + 主控制）
3. 信息区（标题 / 来源 / 元数据 / 简介）
4. 操作区（下载 / 保存 / 导出 / 分享 / 更多）
5. 状态区（下载 / 导出 / 错误恢复）

关键要求：
- 主内容优先
- 次要操作折叠到 menu/sheet
- 状态清晰但不抢主视觉

### 6.4 Search
- 作为一级能力建设
- 最近搜索 / 建议 / 筛选 / 统一结果列表样式

### 6.5 Settings
- 分组清晰
- 项目统一
- 文案简洁

## 8. 状态系统蓝图

### 页面级状态
- Initial
- Loading
- Empty
- Error
- Loaded
- Refreshing

### 项目级状态
- Idle
- In Progress
- Success
- Failed
- Unsupported

### 反馈规范
- inline：列表项状态、空状态、可重试错误
- snackbar：短期确认反馈
- dialog/sheet：危险操作确认、多动作选择、阻塞错误

## 9. Bug 修复主线

### 必修类型
1. 编译与类型问题
2. 布局溢出
3. 参数丢失 / 导航错误
4. 状态不同步
5. 操作后反馈缺失
6. 下载/导出/播放链路异常表现
7. 长文案 / 小屏 / 边界数据 UI 崩坏

### 优先级
- P0：崩溃、编译失败、主流程断裂
- P1：状态错误、反馈混乱、布局溢出、列表与详情不一致
- P2：文案、次要页面不统一、细节动效
