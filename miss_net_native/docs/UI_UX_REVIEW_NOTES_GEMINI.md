# Gemini 审阅要点摘要

## 最关键修正

1. Home 不要继续停留在“高效率入口”的抽象表述，而要明确成“任务恢复入口”
2. Library / Downloads 不只是资产中心，更是任务与资产状态中心
3. Player 首轮不要过度扩张，只做轻量收口
4. 状态系统首轮要收敛，避免组合爆炸
5. 动画策略要克制，避免偏离 GDrive 式成熟感

## 值得吸收的好点子

- Recent Zone：最近访问 / 继续播放 / 最近完成任务
- 资产流思维：用户关心的是“离线可用”，而不是内部下载/导出步骤
- 错误恢复优先做可理解文案 + 一键恢复
- 列表项要支持“扫一眼就懂”

## 不应过度设计

- 不把 Home 做成新的展示页
- 不把 Search 做成复杂专业工具
- 不在首轮把 Settings 做得过重
- 不在首轮做太多动画和共享元素过渡
